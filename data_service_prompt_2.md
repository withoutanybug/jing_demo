# About 页面 — GA4 数据能力展示模块

## 定位
展示"我具备 Google Analytics 4 全链路数据能力"，所有数据为演示。

## 技术约束
- 纯HTML+CSS，无JS图表库
- 模拟GA4后台界面用CSS画
- 表格用原生table
- 数据写死

## 模块结构（从上到下）

### 模块1：能力标签
横向排列5个标签：
- GA4 事件埋点
- 转化漏斗设计
- 用户路径分析
- 自定义维度
- Looker Studio 报表

样式：10px字号，大写，字间距0.1em，背景var(--color-accent)，白色文字，圆角4px，padding 4px 12px

### 模块2：服务流程（4步骤）
横向4步流程图，用CSS实现：
1. 账号配置 → 2. 事件埋点 → 3. 转化追踪 → 4. 报表搭建

每步：圆形数字 + 标题 + 一行描述
用箭头连接

### 模块3：模拟GA4后台界面
画一个看起来像GA4后台的卡片：
- 顶部：Google Analytics 4 标题 + "演示数据"红色标签
- 左侧导航：首页/报告/探索/配置（首页高亮）
- 右侧内容：
  - 2个指标卡片：用户数 1,247 / 互动率 68.3%
  - 下方一个迷你折线图（SVG polyline，上升趋势）

配色参考GA4：蓝色#1a73e8，灰色背景#f8f9fa

### 模块4：埋点事件设计表
原生table，展示6个事件：

| 事件名称 | 触发条件 | 参数 | 业务意义 |
| page_view | 页面加载 | page_location | 基础流量 |
| nav_click | 点击导航 | page_name | 兴趣分布 |
| image_click | 点击图片 | image_id, category | 内容吸引力 |
| scroll_depth | 滚动50%/75%/90% | percent_scrolled | 完读率 |
| filter_click | 点击筛选 | category_name | 用户偏好 |
| contact_submit | 提交表单 | form_id | 转化追踪 |

表头：背景var(--color-ink)，白色文字
行：交替白色/#fafaf8

### 模块5：数据洞察案例
一个卡片，包含：
- 标题：数据洞察示例（演示）
- 发现：Works页面停留3'12"，Contact仅1'20"
- 假设：用户被内容吸引，但联系意愿未被引导
- 行动：增加"预约咨询"CTA
- 结果：转化率提升24%（演示）

每行前加小圆点或箭头符号
